## Problem

The `using-superpowers` branch carries 12 commits that constitute the
v1.2.0-harness rebuild — the SDK-shape hook contract, `runHarness`
collapse of the three ReAct loops, parallel readonly tool batches,
fail-closed MCP auth, and the closure-doc deliverables (ADRs 007-010 +
`docs/PARITY.md`). PR #21 (`review/using-superpowers-fixes`) already
landed on the current branch but contained only review-fixup commits,
not the rebuild itself. None of the 12 substantive commits are
ancestors of `fix/post-merge-review-cycle-1`.

The current branch additionally carries three cycles of post-merge
correctness fixes (`2fc2c0f`, `ab4f34b`, `9a72133`) — RLS fix in
`source-cache`, `post_turn` in `finally`, surrogate-safe truncation,
under-load hard-fail, turn-delta accounting, USD constant. These are
real fixes that must survive the port.

The task: bring the v1.2.0-harness work onto the current branch in a
form where the closure docs (`docs/adr/010-deferred-phases.md` and
`docs/PARITY.md`) are *truthful* — every "implemented" row in PARITY.md
must point to code that exists on the resulting branch.

## Non-goals

- **Phases 6, 8, 9, 11 stay deferred** per ADR 010: no permission
  resolver, no testcontainer integration suite, no Langfuse hot-path
  spans, no scenario parity harness.
- No drive-by refactors outside the harness rebuild surface.
- No rebase / history-rewrite of `using-superpowers`. Cherry-pick only.

### Doc-truth pass

After all 12 cherry-picks land and tests pass, walk the closure docs
against the resolved code:

1. **`docs/PARITY.md`**: every row marked `implemented` must have a
   live file/symbol on disk. Rows that became `partial` or moved status
   during conflict resolution get updated.
2. **`docs/adr/007-hook-system-rebuild.md`**: hook count claims
   ("16 hook points", "10 hook names in `BUILTIN_REGISTRARS`") must
   match the resolved `BUILTIN_REGISTRARS.size` and the count of valid
   `lifecycle:` enum values. Patch numerics; do not rewrite the
   rationale.
3. **`docs/adr/008-collapsed-react-loop.md`**: route enumeration must
   match resolved `services/agent-claw/src/routes/*.ts`.
4. **`docs/adr/009-permission-and-decision-contract.md`**: hook
   examples cited must compile against the resolved
   `core/hook-output.ts`.
5. **`docs/adr/010-deferred-phases.md`**: no edits expected — it is a
   forward-looking deferral record.
6. **`CLAUDE.md` Harness Primitives section**: the table of 16 hook
   points and `BUILTIN_REGISTRARS` story must match resolved code.
   Test counts in the "Test counts (current branch)" section get
   re-run and updated.

## Out-of-scope

- Modifying `using-superpowers` itself.
- Closing PR #21 or amending its history.
- Implementing any v1.3 deferred phase (Phase 6 permission resolver,
  Phase 8 testcontainer suite, Phase 9 Langfuse spans, Phase 11
  scenario parity harness).
- Tagging `v1.2.0-harness` — that's a follow-up release-process step,
  not part of the port.

## Acceptance criteria

A reviewer reading the resulting branch finds:

1. All four ADRs (007-010) live under `docs/adr/`.
2. `docs/PARITY.md` exists; every "implemented" row points to extant code.
3. `services/agent-claw/src/core/hook-output.ts` exists; hook
   implementations return `HookJSONOutput`-shaped values; the
   lifecycle aggregates with `deny>defer>ask>allow` precedence.
4. `services/agent-claw/src/core/streaming-sink.ts` (or equivalent)
   exists; `runHarness` is the single ReAct loop reachable from
   `/api/chat`, `/api/chat/plan/approve`, `/api/sessions/:id/plan/run`,
   `/api/sessions/:id/resume`, `/api/deep_research`.
5. `services/agent-claw/tests/integration/all-hooks-fire.test.ts`
   exists and passes.
6. `MCP_AUTH_REQUIRED` defaults to fail-closed; a missing token
   without `MCP_AUTH_DEV_MODE=true` returns 401.
7. The three post-merge correctness fixes (RLS user-id pass-through,
   `post_turn` in `finally`, surrogate-safe truncate) are observable
   in the resolved code.
8. CLAUDE.md "Harness Primitives" + "Test counts" sections match
   resolved reality.
