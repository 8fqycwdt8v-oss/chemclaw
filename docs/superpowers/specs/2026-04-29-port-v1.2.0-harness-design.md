## Problem

The `using-superpowers` branch carries **20 commits** ahead of `main`
that constitute (a) the v1.2.0-harness rebuild proper and (b) work that
implements the four phases ADR 010 originally said were deferred. PR
#21 (`review/using-superpowers-fixes`) already landed on the current
branch but contained only review-fixup commits, not the rebuild itself.
None of the 20 substantive commits are ancestors of
`fix/post-merge-review-cycle-1`.

**Inventory:**
- Commits 1–12 (`cb0b35b → ecc2fc0`): v1.2.0-harness rebuild — SDK-shape
  hook contract, `runHarness` collapse of three ReAct loops, parallel
  readonly tool batches, fail-closed MCP auth, ADRs 007–010, PARITY.md.
- Commits 13–15 (`38c06ef → 9a6f90b`): security/correctness fixes on top
  of v1.2 — `redact ask_user`, AbortSignal `Promise.race`, doc realignment.
- Commits 16–17 (`73c93ac`, `0e8c1cc`): **Phase 6** — permission resolver
  (`core/permissions/resolver.ts`), `permissionMode`, `allowedTools`,
  `disallowedTools`, workspace-boundary validation
  (`security/workspace-boundary.ts`).
- Commit 18 (`5247f8d`): **Phase 8** — testcontainer-driven integration
  tests for etag conflict, chained execution, reanimator round-trip.
- Commit 19 (`f20103e`): **Phase 9** — per-hook + per-tool OTel spans on
  hot paths.
- Commit 20 (`c0c949e`): **Phase 11** — scenario-driven mock parity
  harness with 8 scenarios.

The current branch additionally carries three cycles of post-merge
correctness fixes (`2fc2c0f`, `ab4f34b`, `9a72133`) — RLS fix in
`source-cache`, `post_turn` in `finally`, surrogate-safe truncation,
under-load hard-fail, turn-delta accounting, USD constant. These are
real fixes that must survive the port.

**Critical doc-truth gap:** `docs/PARITY.md` and
`docs/adr/010-deferred-phases.md` at HEAD of `using-superpowers`
**still claim Phases 6/8/9/11 are deferred**, even though commits 16–20
on the same branch implement them. The task is therefore not just to
port code but to **reconcile docs with the post-port reality**:

- Flip Phase 6 / 8 / 9 / 11 rows in PARITY.md from `deferred` → `implemented`.
- Rewrite ADR 010 to reflect that those phases landed in the same
  release, leaving only genuinely-deferred items (v1.4+ rows: setting
  sources, ToolSearch, effort levels) as forward-looking.
- ADR 010's status changes from "list of deferrals" to "release-notes
  retrospective + remaining v1.4 deferrals".

After the port, every "implemented" row in PARITY.md must point to code
that exists on the resulting branch.

## Non-goals

- v1.4+ items (setting sources, ToolSearch lazy loading, effort levels,
  cost-correct streamed-text refactor) stay deferred.
- No drive-by refactors outside the harness rebuild surface.
- No rebase / history-rewrite of `using-superpowers`. Cherry-pick only.
- Releasing / tagging `v1.2.0-harness` (or whatever supersedes that
  label given the expanded scope) — that's a follow-up release-process
  step.

### Verification gates

A port is not "done" until all gates pass *in the worktree*:

- `cd services/agent-claw && npm test` — green; the canaries are
  `tests/integration/all-hooks-fire.test.ts`,
  `tests/integration/chat-streaming-via-harness.test.ts`,
  `tests/integration/pre-compact-end-to-end.test.ts`,
  `tests/parity/parity.test.ts` (8 scenarios), and the testcontainer
  trio (`etag-conflict`, `chained-execution`, `reanimator-roundtrip`).
- Testcontainer-gated tests need Docker. If Docker is unavailable,
  they should self-skip rather than fail; verify the helper at
  `services/agent-claw/tests/helpers/postgres-container.ts` enforces
  this. Skipping is acceptable for the port verification step; the
  CI gate runs them.
- `cd services/agent-claw && npx tsc --noEmit` — clean.
- `cd services/paperclip && npm test` — green (regression check;
  shouldn't change).
- `python3 -m pytest tests/unit/test_redactor.py
  services/mcp_tools/common/tests/

## Out-of-scope

- Modifying `using-superpowers` itself.
- Closing PR #21 or amending its history.
- Implementing any v1.3 deferred phase (Phase 6 permission resolver,
  Phase 8 testcontainer suite, Phase 9 Langfuse spans, Phase 11
  scenario parity harness).
- Tagging `v1.2.0-harness` — that's a follow-up release-process step,
  not part of the port.

## Acceptance criteria

A reviewer reading the resulting branch finds:

1. All four ADRs (007-010) live under `docs/adr/`.
2. `docs/PARITY.md` exists; every "implemented" row points to extant code.
3. `services/agent-claw/src/core/hook-output.ts` exists; hook
   implementations return `HookJSONOutput`-shaped values; the
   lifecycle aggregates with `deny>defer>ask>allow` precedence.
4. `services/agent-claw/src/core/streaming-sink.ts` (or equivalent)
   exists; `runHarness` is the single ReAct loop reachable from
   `/api/chat`, `/api/chat/plan/approve`, `/api/sessions/:id/plan/run`,
   `/api/sessions/:id/resume`, `/api/deep_research`.
5. `services/agent-claw/tests/integration/all-hooks-fire.test.ts`
   exists and passes.
6. `MCP_AUTH_REQUIRED` defaults to fail-closed; a missing token
   without `MCP_AUTH_DEV_MODE=true` returns 401.
7. The three post-merge correctness fixes (RLS user-id pass-through,
   `post_turn` in `finally`, surrogate-safe truncate) are observable
   in the resolved code.
8. CLAUDE.md "Harness Primitives" + "Test counts" sections match
   resolved reality.

   Adds new files; minimal conflict surface. Run `permission-mode`
   unit tests as immediate canary. Run full tests.

   testcontainer infra, observability spans, parity harness. Run
